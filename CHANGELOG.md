# Changelog

## [Unreleased]

### Added
- **79 new lessons from a backfill re-mine — the lesson library goes 428 → 507, and the two non-Claude-Code tools stop being ghosts: Claude Design 8 → 26, Codex ~2 → 37.** Every source transcript was *already* in `.processed.json`; they had simply never been mined for design or Codex content, because the extraction scope excluded it (see the Fixed entry below). The seam wasn't thin, it was untouched — a 28k-word design course, a 66-minute web-design video, a 42-video Codex course section, and the Claude-Code/Codex intersection videos, ~180k words re-read by six parallel miners returning 83 candidates. Cross-miner dedupe merged two independently-found duplicate claims; 79 written. **17 lessons are cross-tool** (`tool:` naming two products) — 21% of the tagged corpus, and almost all of them about the *seam*: handoffs, porting, routing between tools. Highlights: Claude Design can't load your custom skills (bracket it instead), its question round times out into defaults and silently spends quota, a whole-repo attach burned 29% of a weekly limit, a Codex project is a working directory rather than a sandbox (it reused another project's API key unasked), Codex writes 2-5x fewer output tokens but has ~6 hook events to Claude Code's ~30 and never auto-spawns a subagent, and `done when` is the load-bearing quarter of a Codex prompt. Miners reported 10 zero-yield transcripts honestly and rejected several confidently-given creator claims that contradicted better-sourced existing lessons. New lessons private by default; public set stays frozen at 197.
- **A `tool:` axis on lessons — the missing dimension, added as a field rather than a folder.** Lessons carry `tool: claude-code | claude-design | codex` (plus optional `about:` for the tool being driven). `category` stays the **topic** and keeps owning the folder tree; `tool` is the **product** and is orthogonal, so the 17 topical categories are unchanged and **no lesson files move**. Absent `tool:` means `claude-code`, which makes the entire pre-existing corpus correct as-is — no migration, and the path-based public allowlist and its frozen public set are untouched. Crucially the field **accepts a list**: `tool: [claude-code, codex]` for genuinely cross-tool lessons, which a tool-keyed folder tree could not express without duplicating. That matters more than it sounds — 69% of Claude Design transcripts are also Claude-Code-heavy (the export→code handoff is inherently two-tool) and 40% of Codex-heavy ones are too (running them side by side, migrating projects, routing between them). Tool means *where the reader types*, not what gets invoked, so calling Codex from Claude Code is `tool: claude-code` + `about: codex`.
- **`lessons/INDEX.md` is now grouped by tool** (`## Claude Design` / `## Codex` / `## Cross-tool` / `## Claude Code`, each sub-grouped by category). The index — not the directory tree — is the browsable per-tool view: a Claude Design lesson living in `workflows/` is listed under `## Claude Design` and links into its real topical folder. `/optimise` scopes an area by `tool:` and uses `focus:` only to narrow within it.
- Since the extraction criteria changed, a pre-2026-07-17 `.processed.json` entry is no longer evidence a transcript's design/Codex content was ever considered — the `all` flag's docs now say so, `/optimise` is told to report an empty tool reconciliation as "not yet mined for X" rather than as a fact about the material, and a backfill re-mine over those scopes is the follow-up.
- **The self-referential entry now tracks its own Codex area.** `projects/claudex-setup/` migrated from the flat layout into two tool sub-areas (`claude-code/`, `codex/`) now that the repo has an `AGENTS.md` of its own to audit. The Codex area was scaffolded **and inventoried** in the same pass rather than left as an empty placeholder: `AGENTS.md` present, no project `.codex/`, no trusted-project row of its own, and a rich global `~/.codex/config.toml` but no global `~/.codex/AGENTS.md`. Its starter backlog is derived from that inventory; the first `lessons/` reconciliation pass is still pending. Audit log gains one row per sub-area.

### Fixed
- **Stale rationale: Claude Design's separate weekly quota is reportedly gone.** Its July 2026 upgrade moved it onto normal plan usage, which silently underpinned most June-era design advice — including "export to Claude Code so you're burning a different session", a trick that dodges nothing once both draw on one pool. `hand-off-claude-design-exports-to-claude-code` had that as its opening premise; amended rather than superseded, since the technique still stands on code-level changes, custom logic and deploy. Three other lessons that leaned on the separate pool got REVISIT caveats. Tagged REVISIT throughout per the stale-workaround policy: **single-source and uncorroborated** — verify in the usage panel before relying on either version.
- **`/extract-lessons` was silently mining only one of the three tools this repo tracks.** It defined a lesson as "an improvement to how someone uses **Claude Code**", listed only Claude Code surfaces as in-scope, and offered 17 categories with no home for Claude Design or Codex technique — so that material was dropped as out-of-scope while the transcripts were still ledgered in `.processed.json` as done. The failure was self-confirming: `/optimise --design` and `--codex` scope by `focus:` lists that mirror `lessons/` folder names, so with no folder to find, both kept concluding "no relevant lessons exist" — and that zero read as a fact about the material rather than a gap in the pipeline. The evidence: ~20 Claude Design transcripts and a comparable Codex set were all marked processed yet yielded ~0 lessons, including a 28k-word Claude Design course (61 mentions of "design system", 18 of "token") and a 1-hour Codex course. Scope is now Claude Code + Claude Design + Codex, the in-scope list names each tool's surfaces, and a zero-lesson result on a technique-dense transcript is called out as a red flag to re-read rather than a valid outcome. Model-comparison videos are also no longer blanket-dropped: a comparison that yields a concrete routing rule is now a lesson.
- **`--design` audits scoped by a `focus:` list that mirrored no real folder.** The design area's focus (`design-systems`, `ui-generation`, `handoff-to-code`) matched no `lessons/` directory, guaranteeing an empty reconciliation regardless of the corpus. Areas now select their lessons by the `tool:` field and use `focus:` only to narrow within it, so an audit can no longer be silently scoped to nothing.
- `/optimise`'s audit-log format example cited `claudex-setup` as a *flat* project — no longer true now that it has tool sub-areas. Repointed to a genuinely flat entry.
- **Submission-readiness pass over the public-facing docs** (six-agent audit ahead of listing the repo): a `SECURITY.md` link pointed at `scripts/publish.sh`, which is stripped from the public mirror, so it 404'd for visitors — de-linked to inline code; the README showcase line undercounted the shipped lessons (~100 → ~200, actual 197); the `/optimise` usage note in the USER_GUIDE omitted `--codex`; a lesson-extraction cron example used the wrong working directory; the README Python floor (`3.x`) disagreed with `pyproject.toml` (`>=3.14`); `pyproject.toml`'s own `version` lagged at `0.1.0` while the plugin/marketplace/CHANGELOG were `0.3.0`; and a stale `[[wikilink]]` pointed at a renamed lesson id. Links, artifacts, and the leak gate otherwise verified clean.
- **Hardened the publish leak-gate against capitalized private identifiers.** The verify step matched patterns case-sensitively, so a title-cased form of a private name (the way such names are usually written in prose) would slip both the scrub and the gate. Patterns now fold their own case via character classes; the one AWS-product exception is kept lowercase-only by design so it does not false-fail the legitimate public lesson that mentions it. The stripped trees and scrubs already kept the current snapshot clean — this closes the gap for future edits.

## [0.3.0] - 2026-07-17

The claudex release — Codex joins Claude Code, and the lessons library grows past 400.

### Added
- **`AGENTS.md`** — the repo's own Codex entry point, the counterpart to `CLAUDE.md` (Codex reads `AGENTS.md` the way Claude Code reads `CLAUDE.md`). Deliberately thin: it defers to `CLAUDE.md` as the single source of truth for the shared working rules (Conventional Commits, CHANGELOG discipline, validation) rather than restating them, since a second copy would drift — with one deliberate exception, a short safety net mirroring only the rules where a miss is catastrophic and unrecoverable (never commit secrets, the publish leak-gate, no private project names in `CHANGELOG.md`), because Codex reads `AGENTS.md` but does not automatically read `CLAUDE.md`. `CLAUDE.md` wins any disagreement. Adds the Codex-specific notes — this repo's Codex config surface, the `projects/<slug>/codex/` tracking sub-area, the `publish.sh` `ALLOWED` allowlist gotcha for new top-level files, and an honest statement of scope. Added to `publish.sh`'s `ALLOWED` list so it ships to the public mirror.

- **`/optimise <project> --codex`** — a third tool sub-area (alongside `--design`) that audits a project's OpenAI Codex setup: it inventories the Codex config surface (`AGENTS.md`, `.codex/`, `~/.codex/config.toml`) like the Claude Code area, reconciles Codex-relevant `lessons/`, and gets its own `habits.md` (Codex is a CLI). A starter Codex sub-area was scaffolded for the first multi-tool project; documented in `projects/README.md` + the audit log.
- **24 new lessons** this cycle: 11 from the Jul 12–16 transcript fetch across 7 creators, 7 from `/extract-social-lessons` (4 verified Claude Code CHANGELOG gotchas — `--forward-subagent-text`, `autoMode` moving out of `settings.local.json`, silent-no-op `Write(path)`/`Glob(path)` permission rules, `AskUserQuestion` idle timeout — plus Willison/Wu/Husain posts), and 6 from a re-pull of the Agentic Coding School course (now 326/349 transcripts via its MCP, matched by title after the course outline was reordered). One supersedes the "5-minute prompt-cache" lesson with the corrected ~1-hour-on-subscription window. New lessons private by default.
- **5 new lessons** from the 8 pending Jul-10 transcripts (a new creator, `simon-scrapes`, plus austin-marchese and sean-kochel; four Fable-5-vs-GPT-5.6 model-comparison videos yielded nothing in scope): `/goal`'s ~4k-char limit (reference a spec file instead of pasting), a privacy "quiet mode" env-var bundle for confidential work, clone-the-real-reviewer skills, prove-a-process-as-a-hand-run-skill-before-automating, and a max-effort-plateaus-at-high finding that was **merged** with the existing `effort-control-for-token-intensive-work` lesson into one balanced model-selection lesson superseding it (keeps the match-effort-to-risk core, corrects the top end: high not max on capable models). ### Changed
- **Rebalanced the `claudex` framing to match reality** — the repo was named for Claude Code **+ Codex** but read as Claude-Code-only, with no Codex entry point of its own. The README tagline/intro now cover both tools and link Codex, the "Why claudex?" note points Codex users at `AGENTS.md`, and the plugin + marketplace manifests describe the `/optimise` Claude Code / Claude Design / Codex audit surface (keywords now include `codex`, `agents`). Paired with an explicit **honest-scope** statement: the lessons library is Claude-Code-first and the commands/subagents/hooks are Claude Code artifacts — Codex is first-class in what the repo *audits and tracks*, not a claim of tool parity.
- **`/optimise` now persists in-flight suggestions to a per-run scratchpad** so stopping a run mid-walkthrough no longer loses freshly mined suggestions. After reconciling lessons, the command flushes this run's keepers to a transient, gitignored `optimise.md` in the audited sub-area *before* walking them; each item is struck as it's applied, backlogged, added to habits, or dropped, and the file is deleted on a clean finish. A leftover `optimise.md` is a resume signal — the next run picks up its pending items instead of re-deriving them from `lessons/`. Documented in `projects/README.md`, the USER_GUIDE, and the command's field reference + self-check.
- **README now explains the `claudex` name and surfaces the `--codex` audit area** — the intro spells out claude + codex, and the `/optimise` capability row/prose list `--design` (Claude Design) and `--codex` (OpenAI Codex) alongside the default Claude Code audit. The USER_GUIDE multi-tool note and `references/project-structure.md` sub-area comment now include `codex/`.
- **Renamed the project/plugin from `claude-setup` to `claudex-setup`** — it now spans Claude Code, Claude Design, and Codex (claude + codex). Updated the plugin + marketplace `name`, the `homepage`, `publish.sh`'s public remote, the README/USER_GUIDE/CLAUDE.md references, `pyproject.toml`/`uv.lock`, and the self-referential tracking folder (`projects/claude-setup/` → `projects/claudex-setup/`). GitHub repos renamed (GitHub's automatic redirects keep existing clone URLs working); the local working-folder rename is a manual follow-up.
- The three lessons-pipeline / audit commands rewritten as explicit step-by-step procedures with final self-checks, folding in the improvements from a skill-format rebuild while keeping all existing behaviour: `/extract-lessons` gains repo-root resolution, a fresh-start library scaffold, exclusion of non-transcript `.txt` seed files, parallel batched Explore miners for large backlogs (mining still never enters the main context), an exact lesson-file format with `## Related` wikilinks, and incremental INDEX updates that preserve the other run footers; `/extract-social-lessons` gains named per-cluster subagents with search budgets, a configurable recency window, tier/voice filters, a `dry-run` mode, and a verbatim-quote-or-drop bar; `/optimise` gains explicit area/mode resolution (including offering to scaffold a missing design sub-area or register an unknown project), an exact audit-log skeleton, a config-vs-habit routing rule, an exact-preview requirement for apply dialogs, and a registry field reference.
- `publish.sh` now **freezes the public lesson set**: only lessons listed in `scripts/public-lessons.allowlist` (the pre-existing ~197) ship to the public mirror, with a frozen public `INDEX.md` + `.processed.json`; newly extracted lessons are private by default, so the public set stays a showcase of `/extract-lessons` output without growing.
- `/optimise` now maintains a central `projects/audit-log.md` tracker — force-created on first use and rewritten each run — so "when was each project last optimised" lives in one command-maintained place instead of drifting across per-project frontmatter and the registry README.
- `/optimise` now supports **multi-tool projects**: a tracked project's folder can hold per-tool sub-areas (`claude-code/`, `claude-design/`), each with its own `current.md` / `improvements.md` / `applied-improvements.md`. `/optimise <project>` audits the Claude Code area (unchanged behaviour); the new `--design` flag audits the Claude Design area — a hand-maintained snapshot, since Claude Design has no `.claude/` to inventory (and no `habits.md`, a CLI-only concept). Single-tool projects keep the flat layout unchanged; the audit log gets one row per sub-area, keyed `<slug>/<tool>`. Documented in the USER_GUIDE and README.

### Fixed
- Transcript fetcher now defeats YouTube's per-IP caption-endpoint throttle by replaying a real Chrome TLS handshake: `fetch_transcripts.py`'s HTTP client swapped from plain `requests` to `curl_cffi` with `impersonate="chrome"`. Added `curl-cffi` to the dev deps (`pyproject.toml`, `uv.lock`) and the `.venv-linux` create command. Applied after pacing alone stopped clearing the flag (three consecutive zero-fetch runs); smoke-tested clean on the throttled IP. See `references/transcript-fetch-throttling.md`.
- Transcript titles/creators scraped from YouTube's embedded JSON are now decoded (JSON string escapes + HTML entities) before use, so filenames and `index.md` show real characters instead of raw escape sequences (e.g. an ampersand rendered as a literal backslash-u-0026).

- A curated `CLAUDE.md` plus reference docs covering integrations, structure, and hook gotchas
