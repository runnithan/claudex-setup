# Hand Off Claude Design Exports to Claude Code for Code-Level Iteration

## TL;DR

Claude Design and Claude Code have separate quotas and strengths; when you hit Design's limit or need code-level changes, export the project as a zip and continue in Claude Code.

## Why it matters

A design-to-deploy build can span both tools — Design for fast visual generation, Code for cheaper code edits, integrations, and shipping. Exporting the zip lets you keep iterating without burning Design quota and unlocks everything Design can't do (custom logic, deploy).

## How to apply

In Claude Design, download the project as a zip and open the extracted folder in Claude Code; tell it what to change, then wire up interactivity/integrations and deploy. Use this as the standard bridge once the visual design is close enough.
