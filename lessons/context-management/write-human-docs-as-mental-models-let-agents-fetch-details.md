# Write Human Docs as Mental Models and Intent, Not Exhaustive Reference

## TL;DR

Humans need mental models and intent; agents can retrieve the specifics. So write documentation that conveys the why and the design decisions, and stop optimizing prose docs for exhaustive completeness.

## Why it matters

Drew Breunig: 'Humans don't require exhaustive documentation, they require mental models. Agents can find the details.' Exhaustive reference is now both unrewarded labor and redundant, while readers who grasp intent 'can reason beyond the obvious.'

## How to apply

Teach the mental model and 'explain the why'; 'detail the design decisions — what did the tool deliberately choose not to do or include?'; then 'don't optimize for completeness... trust the agent to facilitate the rest.' Write skills for the agent and reserve prose docs for human understanding.
